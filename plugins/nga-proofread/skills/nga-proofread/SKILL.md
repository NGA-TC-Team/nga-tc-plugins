# NGA Proofread Skill

잭이 NGA 컨설팅 현장에서 사용하는 작성 스타일을 다른 사람의 글에도 강제·적용하는 스킬.

## 핵심 룰

### 제거
- **면책 조항**: "도움이 되었으면", "참고만", "정답은 아닐 수 있지만"
- **사과·후회**: "죄송하지만", "혹시 도움이 안 된다면"
- **충전재**: "정말로", "사실은", "솔직히", "어떻게 보면"
- **과도한 이모지**: 의미 전달에 필수가 아닌 것 전부
- **자명한 영어 병기**: "전략(strategy)", "분석(analysis)" 같은 표기

### 유지
- 수치, 날짜, 회사명, 사람명은 절대 임의 변경 금지
- 추상적 표현 → 구체 액션 (예: "검토 필요" → "수요일까지 DK가 검토")

### 한국어 미세 룰
- "~인 것 같습니다" → "~입니다" (확신 있을 때)
- "~할 수 있을 것입니다" → "~할 수 있습니다"
- "~을 통해서" → "~으로"

## 산출물별 톤

| 상황 | 톤 |
|---|---|
| 임원 보고서 | 결론 먼저, 표·수치 중심 |
| 슬랙 메시지 | 1-3문장, 액션·기한·소유자 명시 |
| 고객사 이메일 | 정중하되 면책 없음. "확인 후 회신"보다 "수요일 18시까지 회신" |
| 교육 자료 | 비유 활용 가능, 핵심 정의는 명료하게 |
| 제안서 | 고객 가치 → 방법론 → 일정·비용 순 |

## 처리 절차

1. 산출물 종류 추정 (보고서/슬랙/이메일/제안/기타)
2. 룰 적용 (제거·유지·미세 룰 순)
3. 다듬은 결과만 출력. 큰 구조 변경 시에만 짧은 변경 노트 첨부
4. 출력 직전 셀프 체크: 수치 보존, 면책 제거, 한 문장 30자 내외, 액션 명확
