# 컬럼 매핑 규칙

NGA "AI 활용 현황 공유 설문" 응답 시트의 한글 헤더를 표준 키(q1.3, q2.1 …)로 자동 매핑한다.

## 매핑 방법

`scripts/build_aiq_html.py`의 `COLUMN_KEYWORDS` 딕셔너리가 단일 소스. 각 표준 키마다 헤더에 포함되어야 하는 한글 키워드 후보 리스트를 둔다. **부분 일치(in)**로 매칭하며, 같은 헤더가 여러 키에 겹치지 않도록 `MAPPING_ORDER` 순서대로 우선 매칭한다.

## 우선순위(`MAPPING_ORDER`)가 중요한 케이스

다음 키들은 헤더에 같은 부분 문자열을 공유하므로 매칭 순서가 결과를 바꾼다. 스크립트는 이 순서를 강제한다.

- `q3_2` (1순위 산출물 링크) → `q3_1` (1순위 사례) — "1순위" 키워드를 더 좁은 쪽에 먼저 줌
- `q3_4` → `q3_3`, `q3_6` → `q3_5` 도 같은 이유
- `q5_2` (절약되는 시간) / `q5_5` (위 사례의 산출물) → `q5_1`, `q5_3`, `q5_4`

## 매칭 실패 시 동작

- `map_columns()`는 매칭 실패한 표준 키를 stderr로 경고 출력만 한다.
- 해당 키의 셀 값은 빈 문자열로 처리되어 HTML에서 "응답 없음"으로 표시된다.
- 부서별로 q4.3·q5.5·q6.2·q6.3가 빠지는 경향이 있으니 빈 매핑은 정상 케이스로 본다.
