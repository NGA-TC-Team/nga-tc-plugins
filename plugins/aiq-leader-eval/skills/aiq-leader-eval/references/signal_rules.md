# 신호 분류 규칙

각 응답자에게 6가지 정성 신호(🟢/🟡/🔴)를 자동으로 매긴다. 잭(NGA)이 코어비즈 26명에게 매긴 실제 패턴에서 역추출한 규칙이다. **점수가 아니라 패턴 라벨**이라는 점이 핵심 — 리더 점수는 별도로 입력한다.

코드 본체는 `scripts/build_aiq_html.py`의 `sig_tool / sig_freq / sig_case / sig_review / sig_impact / sig_share`. 이 문서는 그 로직을 사람이 읽을 수 있게 풀어놓은 것이며, **두 파일이 어긋나면 코드를 정답**으로 본다.

## 색상 의미

- 🟢 **sig-green** — 정착·체화 — 구체 사례·다양성·일상 흐름이 명확
- 🟡 **sig-yellow** — 진행 — 신호는 있으나 사례 수·구체성·정착도가 한정적
- 🔴 **sig-red** — 시작·간헐 — 사례·정착·공유 흔적 부족

## 신호별 분류 규칙

### 01 툴 (`sig_tool`)
**근거 질문**: q2.1 정기 사용 도구 (복수 응답)

| 도구 수 n | 신호 | 라벨 | 디테일 |
|---|---|---|---|
| ≥ 3 | 🟢 | 폭넓음 | `n종` |
| 2  | 🟡 | 복수   | `n종` |
| 1  | 🔴 | 단일   | `n종` |
| 0  | 🔴 | 없음   | 응답 없음 |

### 01 빈도 (`sig_freq`)
**근거 질문**: q1.3 AI가 함께 작동하는 업무 비중

| q1.3 키워드 | 신호 | 라벨 |
|---|---|---|
| "거의 항상" 또는 "75" | 🟢 | 체화 |
| "자주" 또는 "50"      | 🟢 | 정착 |
| "꾸준히" 또는 "25"    | 🟡 | 진행 |
| "가끔" 또는 "10"      | 🔴 | 간헐 |
| 그 외/공란            | 🔴 | 간헐 |

### 02 사례 (`sig_case`)
**근거 질문**: q3.1 / q3.3 / q3.5 (1·2·3순위 사례)
`n = 비어있지 않은 사례 수`, `avg_len = 비어있지 않은 사례 텍스트 평균 길이(자)`

| 조건 | 신호 | 라벨 | 디테일 |
|---|---|---|---|
| n ≥ 3 AND avg_len ≥ 60 | 🟢 | 풍부 | `n건 / 평균 {avg_len}자` |
| n ≥ 2 또는 (n=1 AND avg_len ≥ 80) | 🟡 | 보유 | 〃 |
| n ≥ 1 | 🟡 | 단편 | 〃 |
| 그 외 | 🔴 | 부재 | 응답 없음 |

### 02 검토 (`sig_review`)
**근거 질문**: q4.1 검토 흐름 / q4.2 오류·편향 수정 사례 / q4.3 체크리스트·템플릿

- `has_checklist`: q4.3 응답이 있고 ["없음", "아직 없음", "X", "x", "없습니다", "없어요", "-", "없다"] 중 하나가 아님
- `has_fix`: q4.2 응답 길이 > 8자
- `has_flow`: q4.1 응답 길이 > 8자

| 조건 | 신호 | 라벨 | 디테일 |
|---|---|---|---|
| has_checklist AND (has_fix 또는 has_flow) | 🟢 | 체계 | 템플릿+검증 흐름 |
| has_checklist 단독                         | 🟡 | 관리 | 체크리스트 보유 |
| has_fix AND has_flow                       | 🟡 | 관리 | 검토 흐름·수정 사례 |
| has_flow 또는 has_fix                      | 🟡 | 의식 | 검토 흐름 인식 |
| 그 외                                       | 🔴 | 부재 | 검토 흐름 약함 |

### 03 임팩트 (`sig_impact`)
**근거 질문**: q5.1 자동화 수 / q5.2 절약 시간 / q5.4 "AI 없으면 못 했을" 사례

- `auto_3plus`: q5.1에 "3" 또는 "6" 포함 (즉 "3–5개" 또는 "6개 이상")
- `time_3plus`: q5.2에 "3", "6", "10" 포함 (3시간 이상)
- `has_no_ai`: q5.4 응답 길이 > 15자

| 조건 | 신호 | 라벨 | 디테일 |
|---|---|---|---|
| auto_3plus AND time_3plus AND has_no_ai | 🟢 | 뚜렷 | `{시간} / {자동화수}` |
| (auto_3plus AND time_3plus) 또는 has_no_ai | 🟡 | 실현 | 〃 |
| time·auto 모두 있고 1+ 포함 | 🟡 | 실현 | 〃 |
| 그 외 | 🔴 | 미미 | 〃 |

### 04 공유 (`sig_share`)
**근거 질문**: q6.1 공유 방식 / q6.2 동료 변화 사례 / q6.3 공유 자료 링크

- `channels = parse_multi(q6.1)`
- `has_no_share`: 채널 중 "공유한 적 없음" 또는 정확히 "없음" 포함
- `n = "공유한 적" 제외한 실제 공유 채널 수`
- `has_change`: q6.2 응답 길이 > 10자
- `has_link`: q6.3에 "http" 포함

| 조건 | 신호 | 라벨 | 디테일 |
|---|---|---|---|
| n ≥ 2 AND (has_change 또는 has_link) | 🟢 | 확산 | `n가지 방식` |
| n ≥ 2 | 🟡 | 시도 | `n가지 방식` |
| n = 1 | 🟡 | 시도 | `1가지 방식` |
| has_no_share | 🔴 | 미공유 | 공유 활동 없음 |
| 그 외 | 🔴 | 미공유 | 응답 없음 |

## 보조 신호

멤버 상세에서 `01 툴` 박스에는 `01 빈도` 신호가, `02 사례` 박스에는 `02 검토` 신호가 보조 신호로 따라붙는다 (예: "🟡 복수 — 2종 · 보조 신호 🟢 체화"). `03 임팩트`와 `04 공유`는 보조 신호 없음. 빌더가 자동 처리.

## 신호 분포로 본 축 블록 색상

축 블록의 좌측 컬러 바(`axis-block.green/yellow/red`)는 그 축 **메인 신호의 색상**을 따른다 — `01축`은 `sig_tool` 색, `02축`은 `sig_case` 색, `03축`은 `sig_impact` 색, `04축`은 `sig_share` 색.

## 코어비즈 26명 분류 결과 (역추출 검증용)

이 규칙으로 코어비즈 응답을 다시 돌려 봤을 때 잭의 원본과 동일하게 나와야 한다. 한 축이라도 어긋난다면 규칙을 수정하기보다 그 응답이 경계 케이스인지 먼저 확인할 것.
