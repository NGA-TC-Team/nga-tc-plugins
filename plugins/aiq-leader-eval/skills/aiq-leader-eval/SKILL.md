# AI-Q 리더 평가 가이드 생성 스킬 (v7)

NGA AX 프로그램에서 사용하는 표준 평가지 v7을 다른 부서에 동일 양식으로 재현하는 스킬. 경영관리 11명 평가지를 기준 양식으로 한다.

> v6 → v7 변경: a2 sub-item 재정의(s24 제거 → ⭐사례 헤더카드 + s21/s22/s23 라벨 변경), a4에 s44 신규(팀 문화 형성, 리더 종합 판단), a1 1-3↔1-4 라벨/순서 조정.

## 입력

1. **설문 응답 파일** — xlsx / csv / tsv (Google Form 익스포트 그대로 사용 가능)
2. **부서명** — 예: "경영관리", "마케팅", "HR", "재무", "퓨처이노베이션"

설문 컬럼이 표준 양식과 동일하지 않을 수 있다. 컬럼 매핑은 `references/column_mapping.md`에 정의된 키워드 기반으로 자동 매칭한다.

## 출력

`{부서명}_AIQ_리더평가가이드.html` — 단일 HTML 파일 (Chart.js·SheetJS CDN 사용). `--output` 미지정 시 입력 파일과 같은 폴더에 저장.

## 표준 워크플로우

작업 순서는 반드시 다음을 따른다.

### Step 1. 입력 파악

설문 파일 경로와 부서명을 확인한다. 부서명이 명시되지 않으면 파일명에서 추론하거나 사용자에게 묻는다. 빌더는 xlsx / csv / tsv 모두 지원한다 — xlsx는 `openpyxl`, csv/tsv는 내장 `csv` 모듈을 쓴다.

### Step 2. HTML 생성

`scripts/build_aiq_html.py`만 호출하면 컬럼 매핑·신호 분류·집계·HTML 빌딩이 한 번에 끝난다. 매핑 규칙은 `references/column_mapping.md`, 신호 분류 규칙은 `references/signal_rules.md`, 4축 sub-item 정의는 `references/axis_schema.md`에 명세되어 있다(스크립트와 1:1 대응).

```bash
python3 scripts/build_aiq_html.py \
  --input "/path/to/{설문파일}" \
  --department "{부서명}" \
  --output "/path/to/{부서명}_AIQ_리더평가가이드.html"
```

`--output`을 생략하면 입력 파일이 있는 폴더에 `{부서명}_AIQ_리더평가가이드.html`로 자동 저장된다.

### Step 3. 결과 제시

생성된 HTML 경로를 사용자에게 알려준다. stderr에 "매핑 실패 컬럼" 경고가 떴거나 경계 케이스가 있었다면 1~2줄로만 짚어준다(긴 보고는 불필요).

## 4축 구조 v7 (변경 금지)

| 축 | 가중치 | Sub-item | 근거 질문 |
|---|---|---|---|

| 03 속도·임팩트 | 30% | s31 자동화 수 / s32 절약 시간 / s33 새 시도 / s34 AI 없으면 못 한 사례 | q5.1 / q5.2 / q5.3 / q5.4·q5.5 |

## 대시보드 6패널

1. 도구 사용 빈도 (정기 사용·복수 응답) — 막대
2. AI 업무 비중 분포 — 도넛 (q1.3)
3. 주 절약 시간 분포 — 도넛 (q5.2)
4. 공유 활동 (복수 응답) — 막대 (q6.1)
5. 가장 큰 어려움 (복수 응답) — 막대 (q8.4)
6. 자동화 보유 건수 — 도넛 (q5.1)

KPI 카드 5종: 응답자 / AI 정기 사용(50%+) / 자동화 보유자(3건+) / 미공유 / 시간 절약 ≥3h/주.

## 신호 색상 코드

- 🟢 sig-green: 정착·체화 — 구체 사례·다양성·일상 흐름
- 🟡 sig-yellow: 진행 — 신호 있으나 한정적
- 🔴 sig-red: 시작·간헐 — 사례·정착 부족

## 점수 저장 (브라우저 localStorage)

- 저장 키: `{부서명}_aiq_v7`
- 멤버별 💾 저장 / 🗑 삭제 버튼으로 평가 확정·취소
- 평가 진행 상황은 액션바에 `입력완료 N / 저장 M / 전체` 형태로 표시
- 📥 엑셀(XLSX) / CSV 내보내기로 보고서 추출

## 참고 파일

- `scripts/build_aiq_html.py` — 입력 파일을 받아 HTML을 생성하는 메인 스크립트
- `references/column_mapping.md` — 설문 컬럼명 → 표준 키 매핑 규칙
- `references/signal_rules.md` — 6가지 신호의 분류 로직
- `references/axis_schema.md` — 4축 sub-item 정의와 evidence question 매핑

## 자주 빠지는 함정

- **컬럼 누락**: 부서별로 일부 질문이 빠질 수 있다. q4.3·q5.5·q6.2·q6.3는 응답률이 낮은 경향. 빈 응답은 "응답 없음"으로 표시하고 점수 영향은 sub-item 단위에서 결정되도록 둔다.
- **s44는 q-evidence 없음**: 4-4 팀 문화 형성은 리더가 4-1·4-2·4-3 응답과 멤버 공유 활동을 종합해 직접 판정한다. 빌더는 evidence 자리에 안내 문구만 출력한다.
- **a1 sub key 순서**: 표시는 1-3에 s14(q2.3 새 도구), 1-4에 s13(q2.2 목적)이다. 데이터 호환을 위해 sub key 자체는 v6와 동일하게 유지하니 라벨만 다르다고 생각하면 된다.
- **이름 중복**: 같은 부서 내 동명이인은 거의 없지만, 이메일이 식별자로 더 안전하다. 정렬은 직책(팀장→파트리더→매니저→MD→AMD→사원) 순.
- **단일 응답자 부서**: 1~2명 부서는 대시보드 차트가 의미 없다. 그래도 동일 양식을 유지하되, KPI 카드만 표시하고 차트는 비어있게 둔다.
- **신호 보조 정보**: 멤버 상세에서 01 툴 옆에 01 빈도, 02 사례 옆에 02 검토가 보조 신호로 따라붙는다(예: "🟡 복수 — 2종 · 보조 신호 🟢 체화"). 03·04는 보조 신호 없음. 빌더 자동 처리.
