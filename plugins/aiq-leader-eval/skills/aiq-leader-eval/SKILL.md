# AI-Q 리더 평가 가이드 생성 스킬

NGA AX 프로그램에서 사용하는 표준 평가지 v6를 다른 부서에 동일 양식으로 재현하는 스킬. 잭(NGA)이 코어비즈 26명 대상으로 설계·검증한 구조를 그대로 유지한다.

## 입력

1. **설문 응답 파일** — xlsx / csv / tsv (Google Form 익스포트 그대로 사용 가능)
2. **부서명** — 예: "코어비즈", "비전비즈", "소호사업부", "마케팅"

설문 컬럼이 코어비즈 양식과 동일하지 않을 수 있다. 컬럼 매핑은 `references/column_mapping.md`에 정의된 키워드 기반으로 자동 매칭한다.

## 출력

`{부서명}_AIQ_리더평가가이드.html` — 단일 HTML 파일 (Chart.js·SheetJS CDN 사용). `--output` 미지정 시 입력 파일과 같은 폴더에 저장.

## 표준 워크플로우

작업 순서는 반드시 다음을 따른다.

### Step 1. 입력 파악

설문 파일 경로와 부서명을 확인한다. 부서명이 명시되지 않으면 파일명에서 추론하거나 사용자에게 묻는다. 빌더는 xlsx / csv / tsv 모두 지원한다 — xlsx는 `openpyxl`, csv/tsv는 내장 `csv` 모듈을 쓴다.

### Step 2. HTML 생성

`scripts/build_aiq_html.py`만 호출하면 컬럼 매핑·신호 분류·집계·HTML 빌딩이 한 번에 끝난다. 매핑 규칙은 `references/column_mapping.md`, 신호 분류 규칙은 `references/signal_rules.md`, 4축 sub-item 정의는 `references/axis_schema.md`에 명세되어 있다(스크립트와 1:1 대응).

```bash
python3 scripts/build_aiq_html.py \
  --input "/path/to/{설문파일}" \
  --department "{부서명}" \
  --output "/path/to/{부서명}_AIQ_리더평가가이드.html"
```

`--output`을 생략하면 입력 파일이 있는 폴더에 `{부서명}_AIQ_리더평가가이드.html`로 자동 저장된다.

### Step 3. 결과 제시

생성된 HTML 경로를 사용자에게 알려준다. stderr에 "매핑 실패 컬럼" 경고가 떴거나 경계 케이스가 있었다면 1~2줄로만 짚어준다(긴 보고는 불필요).

## 4축 구조 (변경 금지)

코어비즈에서 확정된 구조. 부서별로 가중치·sub-item을 변경하지 않는다.

| 축 | 가중치 | Sub-item | 근거 질문 |
|---|---|---|---|
| 01 AI 툴 활용 | 25% | s11 비중 / s12 도구 다양성 / s13 정착 깊이 / s14 신규 시도 | q1.3 / q2.1 / q2.2 / q2.3 |
| 02 결과물 완성도 | 30% | s21 사례 구체성 / s22 검증 흐름 / s23 오류 수정 / s24 체크리스트 | q3.1~q3.6 / q4.1 / q4.2 / q4.3 |
| 03 속도·임팩트 | 30% | s31 자동화 수 / s32 절약 시간 / s33 새 시도 / s34 AI 없으면 못 한 | q5.1 / q5.2 / q5.3 / q5.4 |
| 04 지식 공유 | 15% | s41 채널 다양성 / s42 동료 변화 / s43 자료 흔적 | q6.1 / q6.2 / q6.3 |

총점 = (a1·25% + a2·30% + a3·30% + a4·15%) × 20, 등급 S(85+) / A(70~85) / B(55~70) / C(<55)

## 신호 색상 코드

- 🟢 sig-green: 정착·체화 — 구체 사례·다양성·일상 흐름
- 🟡 sig-yellow: 진행 — 신호 있으나 한정적
- 🔴 sig-red: 시작·간헐 — 사례·정착 부족

## 참고 파일

- `scripts/build_aiq_html.py` — 입력 파일을 받아 HTML을 생성하는 메인 스크립트. 컬럼 매핑·신호 분류·집계·HTML 빌딩을 모두 포함한다.
- `references/column_mapping.md` — 설문 컬럼명 → 표준 키 매핑 규칙
- `references/signal_rules.md` — 6가지 신호의 분류 로직 (코어비즈 26명에서 역추출)
- `references/axis_schema.md` — 4축 sub-item 정의와 evidence question 매핑

## 자주 빠지는 함정

- **컬럼 누락**: 부서별로 일부 질문이 빠질 수 있다. q4.3(체크리스트), q5.5(산출물 링크), q6.2(동료 변화), q6.3(자료 링크)는 응답률이 낮은 경향. 빈 응답은 "응답 없음"으로 표시하고 점수 영향은 sub-item 단위에서 결정되도록 둔다.
- **이름 중복**: 같은 부서 내 동명이인은 거의 없지만, 이메일이 식별자로 더 안전하다. 정렬은 응답 순서 또는 직책(팀장→파트리더→매니저→MD→AMD→사원) 순.
- **단일 응답자 부서**: 1~2명 부서는 대시보드 차트가 의미 없다. 그래도 동일 양식을 유지하되, KPI 카드만 표시하고 차트는 "응답자 수 부족" 메시지를 띄운다.
- **신호 보조 정보**: 멤버 상세에서 각 축마다 보조 시그널(예: "🟡 복수 — 2종 · 보조 신호 🟢 체화")이 같이 나온다. 이건 메인 신호 옆에 보조 축 정보를 한 번 더 보여주는 거다. 빌더 스크립트가 자동 생성한다.
